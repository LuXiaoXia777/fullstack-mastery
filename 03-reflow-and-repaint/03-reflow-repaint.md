# 03. 让视觉动起来：重排与重绘
> "动画不只是位移，更是美感与性能的博弈。"

### 💡 核心感悟 (Quotes)
在 UI 设计中，我们习惯于随心所欲地制作动效；但在浏览器中，每一次像素的变化都有代价。理解“重排”与“重绘”，是区分初级前端和高级全栈的分水岭，也是确保产品流畅度的基石。

### 🎯 学习要点
1. **重排 (Reflow) 是昂贵的**：改变盒子的尺寸、位置会触发重排，就像在 Figma 中调整一个巨大的 Auto Layout 容器，会引起周围所有元素的连锁反应。
2. **重绘 (Repaint) 是中等的**：改变颜色、阴影等视觉属性，不影响位置，代价较小。
3. **合成 (Composite) 是极致的**：利用 `transform` 和 `opacity` 进行动画，直接动用显卡（GPU），不产生重排，极其顺滑。

### 🛠️ 知识点拆解
| UI 动作 (Figma) | 触发级别 | CSS 属性示例 | 性能建议 |
| :--- | :--- | :--- | :--- |
| **调整宽度/高度** | 🔴 重排 (Reflow) | `width`, `height` | 尽量避免在动画中使用 |
| **改变位置** | 🔴 重排 (Reflow) | `top`, `left`, `margin` | 动画时改用 `transform: translate` |
| **修改颜色/阴影** | 🟡 重绘 (Repaint) | `color`, `box-shadow` | 可以接受，但要注意频率 |
| **缩放/透明度** | 🟢 合成 (Composite) | `transform: scale`, `opacity` | **首选！** 极其顺滑且不费性能 |
