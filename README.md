# 🚀 全栈学习路径 (Fullstack Mastery Path)
> 这是一个 UI 设计师向全栈开发的进化史。每一项任务被勾选 `[x]` 都代表我离产品闭环更近了一步。

| 学习阶段 | 核心知识点 | 达成目标 (掌握标准) | 建议学习步骤 | 学习难度 | 建议时间 |
| :--- | :--- | :--- | :--- | :---: | :---: |
| **视觉还原与网页骨架** | [ ] **HTML5 语义化** | 能正确使用 `nav`, `main`, `article` 等标签构建无障碍网页。 | 1. 放弃纯 `div`，用语义化标签重构你的 Figma 首页布局。 | ⭐ | 2 小时 |
| | [ ] **CSS 盒模型** | 彻底理解 `padding`, `border`, `margin` 对元素实际尺寸的影响。 | 1. 在 Chrome 开发者工具里手动修改盒模型参数，观察布局变化。 | ⭐ | 3 小时 |
| | [ ] **CSS 优先级** | 搞清楚为什么有些样式写了不生效，解决样式冲突问题。 | 1. 练习使用类选择器、ID 选择器，理解权重分值计算规则。 | ⭐⭐ | 2 小时 |
| | [ ] **重排与重绘** | 能够写出高性能的 CSS 动画，避免页面卡顿。 | 1. 学习哪些属性（如 `transform`）不会触发重排，并在动画中使用。 | ⭐⭐⭐ | 4 小时 |
| | [ ] **关键渲染路径** | 明白浏览器从下载代码到画出像素的 5 个步骤。 | 1. 使用性能面板观察 HTML 解析到屏幕绘制的全过程。 | ⭐⭐⭐ | 5 小时 |
| **编程基础与逻辑大脑** | [ ] **JS 数据类型** | 理解数字、字符串、对象、数组在内存中的存储区别。 | 1. 在控制台练习各种类型转换，掌握常用数组操作方法。 | ⭐ | 5 小时 |
| | [ ] **DOM 事件流** | 搞懂什么是事件冒泡和捕获，实现复杂的交互点击。 | 1. 给一个列表添加点击事件，练习如何精准定位被点击的项目。 | ⭐⭐ | 5 小时 |
| | [ ] **闭包 (Closure)** | 理解函数内部如何访问外部变量，解决变量污染问题。 | 1. 写一个简单的计数器函数，体验私有变量的封装。 | ⭐⭐⭐⭐ | 8 小时 |
| | [ ] **原型与原型链** | 理解 JS 对象是如何继承属性的，掌握底层运行机制。 | 1. 尝试给数组手动添加一个自定义方法，理解 `prototype` 概念。 | ⭐⭐⭐⭐ | 10 小时 |
| | [ ] **异步编程 (Promise)** | 学会处理 AI API 调用时的等待、成功和失败状态。 | 1. 用 `fetch` 调用一个公开 API，练习 `async/await` 写法。 | ⭐⭐⭐ | 10 小时 |
| | [ ] **模块化 (ESM/CJS)** | 学会把庞大的代码拆分成多个文件进行管理。 | 1. 将一个 JS 文件拆分为导出（export）和导入（import）两个文件。 | ⭐⭐ | 3 小时 |
| **现代框架与组件工程** | [ ] **虚拟 DOM** | 理解 React 为什么不需要频繁操作真实 DOM 也能保持高性能。 | 1. 对比原生 JS 和 React 的更新机制，理解“状态驱动视图”逻辑。 | ⭐⭐⭐ | 5 小时 |
| | [ ] **组件生命周期** | 知道在组件“出生”、“改变”、“销毁”时该做什么。 | 1. 在控制台打印组件各阶段的日志，观察数据加载的时机。 | ⭐⭐⭐ | 6 小时 |
| | [ ] **React Hooks** | 掌握 `useState` 和 `useEffect` 两个最核心的逻辑钩子。 | 1. 实现一个自动计时的倒计时器，掌握副作用（Effect）的清理。 | ⭐⭐⭐ | 12 小时 |
| | [ ] **Vue 3 组合式 API** | 学会 Vue 的逻辑复用方式（如果你选择 Vue 路线）。 | 1. 用 `ref` 和 `reactive` 定义响应式数据，实现简单的表单绑定。 | ⭐⭐⭐ | 10 小时 |
| | [ ] **组件通信 (Props)** | 实现组件之间的数据传递（父传子、子传父）。 | 1. 设计一个“父组件按钮触发子组件弹窗”的交互逻辑。 | ⭐⭐ | 5 小时 |
| | [ ] **状态管理 (Redux/Pinia)** | 解决多个页面共享同一个数据（如用户信息）的难题。 | 1. 将登录后的用户名存入状态中心，在所有页面实时同步。 | ⭐⭐⭐⭐ | 15 小时 |
| | [ ] **前端路由 (SPA)** | 实现页面切换时不刷新整个浏览器，像原生 App 一样流畅。 | 1. 使用 `Router` 库配置你的作品集“首页”与“详情页”。 | ⭐⭐⭐ | 8 小时 |
| **底层原理与进阶逻辑** | [ ] **Diff 算法** | 理解 React 是如何计算出哪些地方需要更新的（面试高频）。 | 1. 通过阅读原理文章，画出两个虚拟 DOM 树对比的简易流程图。 | ⭐⭐⭐⭐ | 8 小时 |
| | [ ] **Vue 响应式原理** | 明白为什么你修改了一个变量，页面会自动跟着变。 | 1. 手写一个简单的 `Proxy` 拦截，模拟数据的响应式更新。 | ⭐⭐⭐⭐ | 10 小时 |
| | [ ] **V8 引擎执行原理** | 了解 JS 源代码是如何变成机器能看懂的二进制指令的。 | 1. 学习垃圾回收机制（GC），理解为什么代码写不好会占内存。 | ⭐⭐⭐⭐⭐ | 12 小时 |
| | [ ] **SSR / SSG 渲染** | 学习如何为了 SEO 让搜索引擎更容易搜到你的网站。 | 1. 使用 Next.js 搭建一个简单的静态生成（SSG）作品详情页。 | ⭐⭐⭐⭐ | 15 小时 |
| **工程化工具与工作流** | [ ] **Git 版本控制** | 能够回滚代码、管理分支，像专业团队一样协作。 | 1. 将你的代码 Push 到 GitHub，并练习如何找回之前的版本。 | ⭐⭐ | 8 小时 |
| | [ ] **NPM / Vite 包管理** | 学会安装、升级、管理各种第三方插件库。 | 1. 使用 `pnpm` 快速搭建一个干净的 React 开发环境。 | ⭐ | 3 小时 |
| | [ ] **Webpack / Vite 配置** | 学习如何打包代码，把代码体积压缩到最小。 | 1. 修改构建配置，实现代码混淆压缩和图片自动转 WebP。 | ⭐⭐⭐ | 10 小时 |
| | [ ] **Babel / Polyfill** | 确保你写的现代代码在旧版浏览器上也能跑。 | 1. 了解转译的概念，配置环境支持最先进的 JS 语法。 | ⭐⭐ | 3 小时 |
| | [ ] **ESLint / Prettier** | 让你的代码像你的设计稿一样整齐，符合行业规范。 | 1. 在编辑器中开启自动保存格式化，统一团队代码风格。 | ⭐ | 2 小时 |
| | [ ] **CI/CD 自动化部署** | 实现代码提交后自动触发服务器更新，无需手动上传。 | 1. 配置 GitHub Actions，实现代码提交即部署到 Vercel。 | ⭐⭐⭐ | 10 小时 |
| | [ ] **Source Map** | 在上线后的压缩代码中，如何精准找到报错的那一行。 | 1. 在生产环境下调试 Bug，学会根据 Source Map 定位源码。 | ⭐⭐ | 2 小时 |
| **性能优化与网络深度** | [ ] **跨域 (CORS)** | 解决前端调接口时最常见的“权限拦截”报错。 | 1. 在本地配置代理（Proxy），解决开发环境下的接口跨域。 | ⭐⭐⭐ | 5 小时 |
| | [ ] **浏览器缓存** | 学习如何利用强缓存和协商缓存减少重复请求。 | 1. 通过 Network 面板观察 `304 Not Modified` 状态码。 | ⭐⭐⭐ | 8 小时 |
| | [ ] **防抖与节流** | 优化搜索框、滚动条等高频触发的性能问题。 | 1. 手写一个防抖函数，应用在搜索框输入监听上。 | ⭐⭐⭐ | 5 小时 |
| | [ ] **资源懒加载** | 让你的作品集长图流在滚动到位置时再加载。 | 1. 使用 `Intersection Observer` 实现图片的滚动加载。 | ⭐⭐ | 4 小时 |
| | [ ] **性能指标 (Web Vitals)** | 学习如何用量化的指标（LCP, CLS）评价网站好坏。 | 1. 使用 Lighthouse 跑分，并根据建议修复性能问题。 | ⭐⭐⭐ | 8 小时 |
| | [ ] **内存泄漏 (Memory Leak)** | 找出那些导致页面越用越卡、浏览器崩溃的原因。 | 1. 在控制台 Profiler 中查找没被清理的定时器或全局变量。 | ⭐⭐⭐⭐ | 10 小时 |
| | [ ] **CDN 内容分发** | 让身处全球各地的访客都能飞速打开你的网页。 | 1. 将你的图片资源上传到 OSS，并通过 CDN 进行分发。 | ⭐⭐ | 3 小时 |
| | [ ] **Tree Shaking** | 自动剔除项目中没用到的代码，减小首屏体积。 | 1. 优化第三方库的引入方式（按需加载），观察打包体积变化。 | ⭐⭐⭐ | 5 小时 |
| | [ ] **HTTP/2 & HTTP/3** | 理解现代网络协议是如何多路复用、加速数据传输的。 | 1. 观察请求流水线，理解头部压缩和二进制帧的概念。 | ⭐⭐⭐⭐ | 10 小时 |
| | [ ] **Web Workers** | 实现多线程计算，让复杂的逻辑不阻塞 UI 渲染。 | 1. 将一个耗时的排序算法移出主线程，保持页面滚动流畅。 | ⭐⭐⭐⭐ | 8 小时 |
| | [ ] **PWA 离线应用** | 让你的网页像 App 一样可以安装到桌面并离线访问。 | 1. 配置 Service Worker，让用户在断网时也能看到缓存内容。 | ⭐⭐⭐⭐ | 15 小时 |
