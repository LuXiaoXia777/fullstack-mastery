# 02. 让盒子呼吸：CSS 盒模型与优先级
> "好的设计不仅在于间距的数值，更在于盒子内外边界的逻辑。"

### 💡 核心感悟 (Quotes)
在 UI 设计中，我们通过属性面板调整 Padding 和 Gap；在全栈开发中，我们要理解每一个 HTML 标签都是一个“盒子”。掌握盒模型，是实现像素级还原设计稿的唯一途径。

### 🎯 学习要点
1. **盒模型四要素**：理解内容 (Content)、内边距 (Padding)、边框 (Border)、外边距 (Margin) 的包裹关系。
2. **内外边距分工**：用 `padding` 创造组件内部的呼吸感，用 `margin` 掌控组件间的距离。
3. **优先级控制**：理解样式叠加的权重（标签 vs 类名 vs ID），解决样式冲突的困惑。

### 🛠️ 知识点拆解
| UI 模块 (Figma) | CSS 属性 | 全栈用途 |
| :--- | :--- | :--- |
| Padding (内边距) | `padding` | 控制内容与边框之间的距离，营造组件内部的空间感 |
| Stroke (描边) | `border` | 定义组件的视觉边界，实现霓虹细线等高级质感 |
| Corner Radius (圆角) | `border-radius` | 将矩形盒子转化为药丸形、圆形或各种弧度的 UI 零件 |
| Auto Layout / Gap (间距) | `margin` | 控制盒子与外部环境的距离，决定页面整体的布局节奏 |
| Fill (背景色) | `background-color` | 填充容器底色，配合透明度实现玻璃拟态效果 |
